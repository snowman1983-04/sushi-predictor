# 寿司販売数予測アプリ 仕様書

**プロジェクト名**：sushi-predictor
**目的**：G検定ポートフォリオ・自己学習用
**対象ユーザー**：本人（職場とは切り離す）
**作成日**：2026-04-27

---

## 1. 概要

過去のダミー販売データと気象条件を入力として、ロジスティック回帰および複数モデルにより、指定した日付・曜日・天気での寿司商品別の販売数を予測するWebアプリを構築する。

職場の業務利用は想定せず、データサイエンスの基礎手法を体系的に実装することで、G検定取得に向けた知識の定着とポートフォリオ化を目的とする。

## 4. ダミーデータ仕様

### 4.1 データ規模

- 期間：2年分（730日）
- 商品数：4種類
- 総レコード数：約2,920行（730日 × 4商品）

### 4.2 商品マスタ

| 商品ID | 商品名 | 通常価格 | 特売価格 |
|---|---|---|---|
| P001 | 水木寿司（10貫） | 500 | 500 |
| P002 | 8貫寿司 | 600 | 500 |
| P003 | 10貫寿司 | 600 | 500 |
| P004 | ランチ寿司（特売日のみ） | 500 | 500 |

### 4.3 カラム定義（CSV）

| カラム名 | 型 | 説明 | 例 |
|---|---|---|---|
| date | date | 販売日 | 2024-04-01 |
| product_id | str | 商品ID | P001 |
| product_name | str | 商品名 | 水木寿司 |
| day_of_week | str | 曜日 | Monday |
| is_weekend | bool | 週末フラグ | False |
| is_holiday | bool | 祝日フラグ | False |
| is_pension_day | bool | 年金支給日フラグ（偶数月15日） | False |
| is_sale_day | bool | 特売日フラグ（水曜・週末） | True |
| weather | str | 天気（sunny/cloudy/rainy/snowy） | sunny |
| temperature | float | 平均気温（℃） | 18.5 |
| precipitation | float | 降水量（mm） | 0.0 |
| sales_count | int | 販売数（目的変数） | 22 |

### 4.4 データ生成ロジック

実データに近い特徴を持たせるため、以下のルールで生成する。

**ベース販売数（商品別）**
- P001（水木寿司）：基準12個/日
- P002（8貫）：基準10個/日
- P003（10貫）：基準15個/日
- P004（ランチ）：特売日のみ基準20個/日、それ以外は0

**増減ルール（係数で乗算）**
- 水曜日の特売日：×2.0
- 週末（土日）：×1.5
- 年金支給日（偶数月15日）：×1.8
- 祝日：×1.4
- 雨天：×0.7
- 雪：×0.5
- 気温30℃以上：×0.8
- 気温0℃以下：×0.9

**ノイズ**
- 正規分布（標準偏差=平均の15%）でランダムノイズを加算

**気象データ**
- 弘前市の月別平均気温・降水確率を参考に生成
- 冬季（12-3月）は雪天日の割合を増やす

## 6. モデル仕様

### 6.1 採用モデル

| モデル | 用途 | 特徴 |
|---|---|---|
| ロジスティック回帰 | （カテゴリ予測時に使用）| 解釈性が高い |
| 線形回帰（多重回帰） | 販売数の連続値予測 | 最もシンプル、解釈性高い |
| ランダムフォレスト | 非線形パターン捕捉 | 特徴量重要度が分かりやすい |
| 勾配ブースティング（XGBoost or LightGBM） | 高精度予測 | ベースライン超え用 |

**注記**：販売数予測は本来「回帰問題」だが、ポートフォリオとして「ロジスティック回帰も実装」するため、販売数を「少ない/普通/多い」の3クラスに離散化したバージョンも実装する。これによりG検対策として両手法をカバーできる。

### 6.2 評価指標

**回帰タスク**
- MAE（平均絶対誤差）
- RMSE（二乗平均平方根誤差）
- R²スコア

**分類タスク（離散化版）**
- 正解率
- 適合率・再現率・F1スコア
- 混同行列

### 6.3 交差検証

- 時系列データのため**TimeSeriesSplit**を使用
- 5-fold交差検証
